# AI 코딩 어시스턴트 규칙 (PROMPT_RULES)

이 프로젝트에 참여하는 모든 AI 어시스턴트는 아래 규칙을 반드시 준수해야 합니다.

## 1. 주석 및 문서화 (Documentation)
- **한글 주석 필수**: 모든 코드의 주석은 **한글**로 작성합니다.
- **기능별 주석**: 함수, 클래스, 컴포넌트 등 주요 기능 단위마다 JSDoc/TSDoc 형태의 주석을 작성하여 역할, 파라미터, 반환값을 명시합니다.
  ```typescript
  /**
   * 사용자 정보를 가져오는 함수
   * @param userId 사용자 ID
   * @returns 사용자 객체
   */
  async function getUser(userId: string) { ... }
  ```
- **복잡한 로직 설명**: 코드가 복잡하거나 중요한 비즈니스 로직이 포함된 경우, 줄 단위 또는 블록 단위로 한글 설명을 추가합니다.

## 2. 코드 수정 절차 (Code Modification)
- **수정 전 설명**: 코드를 변경하기 전에, **어떤 내용을 어떻게 수정할 것인지 한글로 설명**해야 합니다.
- **사전 허락**: 설명 후 반드시 **사용자의 허락(승인)**을 받은 뒤에 실제 코드 수정을 진행합니다.
- **변경 이유 명시**: 단순 수정이 아니라 왜 이 변경이 필요한지(버그 수정, 성능 개선, 리팩토링 등) 이유를 함께 설명합니다.

## 3. 커뮤니케이션 (Communication)
- **언어**: 사용자와의 대화는 기본적으로 **한글**을 사용합니다.
- **명확성**: 기술적인 내용은 이해하기 쉽게 풀어서 설명하되, 정확성을 유지합니다.
- **질문하기**: 요구사항이 불명확하거나 모호한 경우, 임의로 판단하지 말고 사용자에게 질문하여 명확히 합니다.

## 4. 프로젝트 표준 (Project Standards)
- **모노레포 구조 준수**: `apps/`와 `packages/`로 나뉜 모노레포 구조를 따르며, 공통 로직은 `packages/shared`, UI는 `packages/ui`를 적극 활용합니다.
- **스타일 가이드**: 프로젝트에 설정된 Lint 및 Prettier 규칙을 준수합니다.
