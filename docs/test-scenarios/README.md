## 공통 작성 기준

- 목적: 어떤 장애를 막기 위한 검증인지 적는다.
- 사전 조건: 계정, 매장, 메뉴, 결제 키, 서버 상태를 적는다.
- 절차: 사용자가 실제로 누르는 순서 또는 API 호출 순서로 적는다.
- 기대 결과: 화면, DB, API 응답, 외부 연동 결과를 함께 적는다.
- 자동화 후보: Playwright 또는 API 테스트로 옮길 수 있는 범위를 표시한다.
